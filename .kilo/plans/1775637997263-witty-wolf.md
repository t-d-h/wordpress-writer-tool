# Plan: Project Detail UI Improvements

## Overview
Improve the project detail view with three enhancements:
1. Display Title/Topic fully (no truncation) and make it clickable to post detail
2. Add WordPress view button that opens the published post URL
3. Replace all action button icons with text labels

## Changes Required

### 1. Backend Changes

#### File: `backend/app/models/project.py`
- Add `wp_site_url` field to `ProjectResponse` model
- This will be populated from the WordPress site data

#### File: `backend/app/routers/projects.py`
- Modify `format_project()` function to include `wp_site_url` from the WordPress site
- Update all endpoints that return project data to include this field

### 2. Frontend Changes

#### File: `frontend/src/components/Projects/ProjectDetail.jsx`

**Change 1: Remove title truncation and make clickable**
- Remove `maxWidth: 250, overflow: 'hidden', textOverflow: 'ellipsis', whiteSpace: 'nowrap'` from title cell
- Wrap title in a clickable link/button that navigates to `/posts/${p.id}`
- Style as a link with hover effect

**Change 2: Add WordPress view button**
- Add new button in Actions column when `p.status === 'published'` and `p.wp_post_id` exists
- Construct URL: `${project.wp_site_url}/?p=${p.wp_post_id}`
- Open in new tab with `target="_blank" rel="noopener noreferrer"`
- Button text: "View on WordPress"
- Style as secondary button

**Change 3: Replace icon buttons with text**
- Replace all `<HiOutlineEye />` with "View"
- Replace all `<HiOutlineArrowPath />` with "Generate"
- Replace all `<HiOutlineRocketLaunch />` with "Publish"
- Replace all `<HiOutlineStop />` with "Unpublish"
- Replace all `<HiOutlineTrash />` with "Delete"
- Update button styling to accommodate text (increase padding if needed)
- Keep conditional logic for when each button appears

**Change 4: Update imports**
- Remove unused icon imports after replacing with text
- Keep only icons used elsewhere (like modal close button)

#### File: `frontend/src/index.css` (if needed)
- Add styling for clickable title links
- Adjust action button padding for text labels if current styling is too tight

## Implementation Steps

1. **Backend - Update ProjectResponse model**
   - Add `wp_site_url: Optional[str] = None` to `ProjectResponse`

2. **Backend - Update projects router**
   - Modify `format_project()` to extract and include `wp_site_url`
   - Test that all project endpoints return the URL

3. **Frontend - Update ProjectDetail component**
   - Remove title truncation styles
   - Add click handler to navigate to post detail
   - Add WordPress view button with URL construction
   - Replace all icon buttons with text labels
   - Clean up unused imports

4. **Testing**
   - Verify title displays fully and is clickable
   - Verify WordPress view button appears for published posts and opens correct URL
   - Verify all action buttons show text instead of icons
   - Verify all existing functionality still works

## Notes

- The WordPress URL construction uses the standard WordPress pattern: `{site_url}/?p={post_id}`
- This assumes WordPress permalinks are set to default; if custom permalinks are used, the URL may need adjustment
- The project already fetches WordPress site data for the name, so adding URL is minimal overhead
- Text buttons will be more accessible and clearer than icons

# Plan: Redesign Login Form to Match Website Design

## Problem
The login form uses custom CSS classes (`login-container`, `login-card`, `error-message`) that are not defined in the design system, resulting in no styling. The form is inconsistent with the rest of the application which uses a cohesive design system with glassmorphism effects, proper spacing, and consistent component patterns.

## Solution
Refactor the Login component to use the existing design system classes and add minimal CSS for login-specific layout needs.

## Changes Required

### 1. Update Login.jsx Component
**File:** `frontend/src/components/Login.jsx`

**Changes:**
- Replace custom classes with design system classes:
  - `login-container` → Use inline styles for centering + `.page-enter` for animation
  - `login-card` → `.card` class
  - `form-group` → Keep (already exists in design system)
  - Add `.form-label` class to labels
  - Add `.form-input` class to inputs
  - Add `.btn-primary` class to submit button
  - `error-message` → `.error-banner` class (if exists) or create consistent error styling
- Add `.page-header` pattern for title
- Add loading state with `.loading-page` and `.loading-spinner` pattern
- Ensure proper semantic HTML structure

**New structure:**
```jsx
<div className="page-enter" style={centeringStyles}>
  <div className="card">
    <div className="page-header">
      <h1 className="page-title">WordPress AI Writer</h1>
      <p className="page-description">Sign in to your account</p>
    </div>

    {error && <div className="error-banner">{error}</div>}

    <form onSubmit={handleSubmit}>
      <div className="form-group">
        <label className="form-label" htmlFor="username">Username</label>
        <input
          id="username"
          className="form-input"
          type="text"
          value={username}
          onChange={(e) => setUsername(e.target.value)}
          required
          disabled={loading}
        />
      </div>
      <div className="form-group">
        <label className="form-label" htmlFor="password">Password</label>
        <input
          id="password"
          className="form-input"
          type="password"
          value={password}
          onChange={(e) => setPassword(e.target.value)}
          required
          disabled={loading}
        />
      </div>
      <button className="btn btn-primary" type="submit" disabled={loading}>
        {loading ? 'Logging in...' : 'Login'}
      </button>
    </form>
  </div>
</div>
```

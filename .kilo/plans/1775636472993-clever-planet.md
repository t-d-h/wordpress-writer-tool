# Plan: Separate Worker Code and Add Comprehensive Logging

## Overview
Two improvements to the worker architecture:
1. Separate worker code from backend code into its own directory
2. Add comprehensive INFO-level logging to see what the worker is doing

## Current State

**Worker Structure:**
- Worker and backend share the same Dockerfile and build context (`./backend`)
- Worker runs with `python -m app.workers.redis_worker`
- Worker code lives in `backend/app/workers/`
- Worker only shows exception logs via `traceback.print_exc()`
- Minimal info logging (only at task start/completion)

**Logging Issues:**
- No structured logging framework
- Only prints on errors and task completion
- No visibility into what's happening during task execution
- No timestamps or log levels

## Changes Required

**Implementation Steps:**

1. Create `worker/` directory structure
2. Move `backend/app/workers/` to `worker/app/workers/`
3. Copy shared modules from `backend/app/` to `worker/app/`:
   - `config.py`
   - `database.py`
   - `redis_client.py`
   - `services/` directory
   - `models/` directory
4. Create `worker/Dockerfile`:
   ```dockerfile
   FROM python:3.11-slim
   WORKDIR /app
   COPY requirements.txt .
   RUN pip install --no-cache-dir -r requirements.txt
   COPY . .
   CMD python -m app.main
   ```
5. Create `worker/requirements.txt` (copy from backend)
6. Create `worker/app/main.py` as worker entry point:
   ```python
   """Worker entry point - imports and runs redis_worker"""
   from app.workers.redis_worker import main
   if __name__ == "__main__":
       import asyncio
       asyncio.run(main())
   ```
7. Update `docker-compose.yml`:
   ```yaml
   worker:
     build:
       context: ./worker
       dockerfile: Dockerfile
     env_file:
       - .env
     network_mode: "host"
     container_name: wp-writer-worker
     command: python -m app.main
     volumes:
       - ./worker/app:/app/app
   ```

### 2. Add Comprehensive Logging

**Logging Configuration:**

Create `worker/app/logging_config.py`:
```python
import logging
import sys

def setup_logging():
    """Configure logging for worker with INFO level and structured output."""
    logging.basicConfig(
        level=logging.INFO,
        format='%(asctime)s | %(levelname)-8s | %(name)s | %(message)s',
        datefmt='%Y-%m-%d %H:%M:%S',
        stream=sys.stdout
    )
    return logging.getLogger(__name__)
```

**Update `worker/app/workers/redis_worker.py`:**

1. Import logging: `from app.logging_config import setup_logging`
2. Initialize logger: `logger = setup_logging()`
3. Replace all `print()` calls with `logger.info()`, `logger.error()`, etc.
4. Add info logs at key points:
   - Worker startup
   - Redis connection
   - Job received
   - Job dispatched
   - Job status updates

**Update `worker/app/workers/tasks.py`:**

1. Import logging: `from app.logging_config import setup_logging`
2. Initialize logger: `logger = setup_logging()`
3. Add info logs for each task:
   ```python
   async def run_research(job_data: dict):
       logger.info(f"[RESEARCH] Starting research for post {post_id}")
       logger.info(f"[RESEARCH] Topic: {topic}")
       logger.info(f"[RESEARCH] Calling AI provider: {provider_id}")
       # ... AI call ...
       logger.info(f"[RESEARCH] Generated {len(research_data)} research points")
       logger.info(f"[RESEARCH] Used {tokens} tokens")
       logger.info(f"[RESEARCH] Completed successfully")
   ```

4. Add progress logs for long-running operations:
   - Before/after AI calls
   - Before/after database updates
   - Before/after image generation
   - Before/after WordPress operations

5. Replace `traceback.print_exc()` with `logger.exception()`

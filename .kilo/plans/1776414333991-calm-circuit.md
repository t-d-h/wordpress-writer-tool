# Plan to fix incorrect status display

The user reported that when a thumbnail generation job is pending, the overall post status is sometimes shown as 'failed', which is misleading. This happens when a previous `publish` job has failed, setting the post status to `failed`, and the user then triggers a new thumbnail job.

To fix this, I will update the post's status to `draft` whenever a new job is queued for a post. This will give the user a clear indication that the post is being processed again.

Here are the steps:

1.  **Modify `backend/app/routers/posts.py`:**
    *   In the `generate_thumbnail` function, update the post's status to `draft`.
    *   In the `generate_outline` function, update the post's status to `draft`.
    *   In the `generate_content` function, update the post's status to `draft`.
    *   In the `publish_post` function, update the post's status to `draft`.

# Plan: Replace Browser Alert with HTML Notification Box

## Overview
Replace the browser `alert()` popup in DefaultModels.jsx with a styled HTML notification box that appears inline on the page.

## Current State
- `DefaultModels.jsx` uses `alert('Default models saved successfully!')` on line 76
- `alert('Error: ' + ...)` on line 78 for error messages
- No existing toast/notification component in the codebase
- CSS has design system variables and `.error-banner` class that could be used as a pattern

## Implementation Plan

### 1. Create Notification Component
**File:** `frontend/src/components/Notification.jsx`

Create a reusable notification component that supports:
- Success and error variants
- Auto-dismiss after configurable timeout
- Manual dismiss button
- Smooth enter/exit animations
- Positioning (top-right or inline)

**Features:**
- Props: `type` ('success' | 'error'), `message`, `duration` (optional), `onDismiss` (optional)
- Uses existing design system colors: `--success`, `--danger`
- Follows existing animation patterns: `fadeIn`, `slideUp`

### 2. Add CSS Styles
**File:** `frontend/src/index.css`

Add notification styles following existing patterns:
- `.notification` - base container
- `.notification.success` - green styling
- `.notification.error` - red styling
- `.notification-icon` - icon container
- `.notification-message` - text content
- `.notification-close` - dismiss button
- Animation classes for smooth transitions

### 3. Update DefaultModels.jsx
**File:** `frontend/src/components/Settings/DefaultModels.jsx`

Changes:
- Import the new Notification component
- Add state for notification: `const [notification, setNotification] = useState(null)`
- Replace `alert()` calls with `setNotification()`
- Render notification component above the form
- Auto-dismiss after 3 seconds on success

**Before:**
```javascript
const handleSave = async (e) => {
  e.preventDefault()
  setSaving(true)
  try {
    await updateDefaultModels(defaults)
    alert('Default models saved successfully!')
  } catch (e) {
    alert('Error: ' + (e.response?.data?.detail || e.message))
  } finally {
    setSaving(false)
  }
}
```

**After:**
```javascript
const handleSave = async (e) => {
  e.preventDefault()
  setSaving(true)
  try {
    await updateDefaultModels(defaults)
    setNotification({ type: 'success', message: 'Default models saved successfully!' })
    setTimeout(() => setNotification(null), 3000)
  } catch (e) {
    setNotification({ type: 'error', message: 'Error: ' + (e.response?.data?.detail || e.message) })
  } finally {
    setSaving(false)
  }
}
```

- Match the dark/light theme support

## Alternative Approaches Considered

### Option 1: Toast Library (react-hot-toast, react-toastify)
**Pros:** Feature-rich, battle-tested
**Cons:** Additional dependency, overkill for simple use case

### Option 2: Context API + Global Toast Manager
**Pros:** Centralized state, accessible from anywhere
**Cons:** More complex setup, not needed for single component

### Option 3: Simple Inline State (Chosen)
**Pros:** Simple, no dependencies, easy to understand
**Cons:** Component-specific (not reusable without refactoring)

**Decision:** Start with Option 3 for simplicity. Can refactor to Option 2 if more components need notifications.

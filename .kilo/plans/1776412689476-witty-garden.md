## 2. Refactor "Projects" Link and Dropdown

- **File:** `frontend/src/components/Sidebar.jsx`
- **Action:** Modify the "Projects" navigation item to be a direct link and have a separate dropdown trigger.

I will replace this:
```javascript
<div className="nav-item" onClick={() => setProjectsOpen(!projectsOpen)}>
  <HiOutlineFolderOpen className="nav-icon" />
  <span>Projects</span>
  {projectsOpen ? <HiOutlineChevronDown style={{ marginLeft: 'auto', fontSize: 14 }} /> : <HiOutlineChevronRight style={{ marginLeft: 'auto', fontSize: 14 }} />}
</div>
```

With this:
```javascript
<div className="nav-item">
  <HiOutlineFolderOpen className="nav-icon" />
  <NavLink to="/projects" end>
    <span>Projects</span>
  </NavLink>
  <span onClick={() => setProjectsOpen(!projectsOpen)} style={{ marginLeft: 'auto', cursor: 'pointer' }}>
    {projectsOpen ? <HiOutlineChevronDown style={{ fontSize: 14 }} /> : <HiOutlineChevronRight style={{ fontSize: 14 }} />}
  </span>
</div>
```

### Changes Explained:

- The outer `div` no longer has an `onClick` handler.
- A `NavLink` is introduced, wrapping the "Projects" `<span>`, to handle navigation to `/projects`. The `end` prop ensures it's only active for the exact `/projects` path.
- The chevron icon is wrapped in a `<span>` with its own `onClick` handler to control the `projectsOpen` state, preserving the dropdown functionality.

This will achieve the desired navigation behavior while keeping the project list accessible.

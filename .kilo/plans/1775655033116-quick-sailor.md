# Plan: Update Create Post Option

## Overview
Enhance the Create Post modal with additional options for publishing, image generation/upload, AI provider model selection, and content length control.

## Changes Required

### 1. Backend Changes

#### 1.1 Update PostCreate Model (`backend/app/models/post.py`)
Add new fields to `PostCreate`:
- `auto_publish: bool = False` - Whether to auto-publish after content generation
- `thumbnail_source: str = "ai"` - "ai" or "upload"
- `thumbnail_provider_id: Optional[str] = None` - AI provider for thumbnail generation
- `thumbnail_model_name: Optional[str] = None` - Model for thumbnail generation
- `section_images_source: str = "ai"` - "ai" or "upload"
- `section_images_provider_id: Optional[str] = None` - AI provider for section images
- `section_images_model_name: Optional[str] = None` - Model for section images
- `target_word_count: Optional[int] = None` - Target word count for content
- `target_section_count: Optional[int] = None` - Target number of sections

#### 1.2 Update Post Router (`backend/app/routers/posts.py`)
- Modify `create_post` endpoint to accept new fields
- Store new fields in MongoDB post document
- Pass new fields to research job via Redis
- Update `format_post` to include new fields in response
- Add `upload_section_images` endpoint for uploading multiple section images at post creation time

#### 1.3 Update Worker Tasks (`worker/app/workers/tasks.py`)
- Modify `run_content` task to use `target_word_count` and `target_section_count` when generating content
- Modify `run_thumbnail` task to use `thumbnail_provider_id` and `thumbnail_model_name` from post document
- Modify `run_section_images` task to use `section_images_provider_id` and `section_images_model_name` from post document
- Modify `run_publish` task to check `auto_publish` flag and only publish if True, otherwise set status to "draft"

#### 1.4 Update AI Service (`backend/app/services/ai_service.py`)
- Modify `generate_outline` to accept and use `target_section_count` parameter
- Modify `generate_full_content` to accept and use `target_word_count` parameter
- Update prompts to include word count and section count constraints

### 2. Frontend Changes

#### 2.1 Update API Client (`frontend/src/api/client.js`)
- Add `uploadSectionImage` function for uploading section images
- Add `uploadSectionImages` function for uploading multiple section images at once
- Update `createPost` to include new fields

#### 2.2 Update ProjectDetail Component (`frontend/src/components/Projects/ProjectDetail.jsx`)
- Add new state for form options:
  - `autoPublish` (boolean)
  - `thumbnailSource` ("ai" or "upload")
  - `thumbnailProviderId` (string)
  - `thumbnailModelName` (string)
  - `sectionImagesSource` ("ai" or "upload")
  - `sectionImagesProviderId` (string)
  - `sectionImagesModelName` (string)
  - `targetWordCount` (number)
  - `targetSectionCount` (number)
  - `availableModels` (array) - for OpenAI-compatible providers
  - `thumbnailFile` (file) - for thumbnail upload
  - `sectionImageFiles` (array) - for section image uploads

- Add `fetchModelsForProvider` function to call `/api/ai-providers/fetch-models`

- Update form UI to include:
  1. **Publishing Options**:
     - Checkbox: "Auto-publish after content generation" (default unchecked)

  2. **Thumbnail Options**:
     - Radio buttons or toggle: "Generate with AI" vs "Upload Image"
     - If "Generate with AI":
       - Dropdown: AI Provider (filter to image-capable providers: gemini, openai)
       - Dropdown: Model Name (based on selected provider)
     - If "Upload Image":
       - File input for thumbnail image
       - Preview of selected image

  3. **Section Images Options**:
     - Radio buttons or toggle: "Generate with AI" vs "Upload Images"
     - If "Generate with AI":
       - Dropdown: AI Provider (filter to image-capable providers)
       - Dropdown: Model Name (based on selected provider)
     - If "Upload Images":
       - File input for multiple section images (accept multiple files)
       - List of uploaded files with previews
       - Note: Images will be assigned to sections in order after outline is generated
       - Alternative: User can also upload section images later in PostView after outline is generated

  4. **Content Length Options**:
     - Number input: Target word count (e.g., 1000, 1500, 2000)
     - Number input: Target number of sections (e.g., 5, 7, 10)

  5. **AI Provider for Content**:
     - Keep existing provider selection
     - When OpenAI-compatible provider is selected:
       - Fetch available models via API
       - Display dropdown with model names instead of text input

- Update form submission handlers to include new fields

#### 2.3 Update PostView Component (`frontend/src/components/Posts/PostView.jsx`)
- Add section for uploading section images if user chose upload option
- Display thumbnail upload option if user chose upload option

### 5. Edge Cases & Considerations

- **Model Fetching**: Handle errors when fetching models from OpenAI-compatible providers (network issues, invalid API key)
- **File Upload**: Validate file types (images only) and size limits
- **Section Image Upload**: Support both upload at creation time and later in PostView. If user uploads at creation, store files temporarily or upload immediately. If fewer images uploaded than sections, handle gracefully.
- **Word Count**: AI may not hit exact word count; provide reasonable range guidance (default 500 words)
- **Section Count**: AI may generate slightly different number of sections; validate and adjust (default 4 sections)
- **Backward Compatibility**: Existing posts without new fields should continue to work
- **Bulk Creation**: Apply same settings to all bulk posts (auto-publish, image sources, word count, section count)
- **Default Values**: auto-publish disabled, AI-generated images, 500 words, 4 sections

### 6. UI/UX Improvements

- Group related options with clear section headers
- Use tooltips or help text to explain each option
- Show loading state when fetching models
- Disable irrelevant options based on selections (e.g., hide model dropdown if provider not selected)
- Provide sensible defaults: auto-publish disabled, AI-generated images, 500 words, 4 sections
- Show preview of uploaded images before submission
- For section images: allow upload at creation (multiple files) or later in PostView (per-section upload)
- For bulk creation: apply same settings to all posts

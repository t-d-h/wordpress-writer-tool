# Plan: Projects Navigation and Delete Confirmation Improvements

## Overview
Two UX improvements to the Projects page:
1. Change sidebar "Projects" from a dropdown to a direct navigation link
2. Replace browser `confirm()` dialog with HTML modal for delete confirmation

## Changes Required

### 1. Sidebar Navigation (`frontend/src/components/Sidebar.jsx`)

**Current behavior:**
- "Projects" is a collapsible section (lines 82-107)
- Shows "All Projects" link and individual project links in dropdown
- Uses `projectsOpen` state to toggle visibility

**New behavior:**
- "Projects" becomes a direct NavLink to `/projects`
- Remove the collapsible dropdown structure
- Remove individual project links from sidebar
- Remove `projectsOpen` state and related logic

**Implementation:**
- Replace the `nav-section` div with a simple `NavLink` component
- Remove lines 88-106 (the dropdown content)
- Remove `projectsOpen` state (line 12-14)
- Keep the "All Projects" link functionality by making "Projects" itself the link

### 2. Delete Confirmation Modal (`frontend/src/components/Projects/ProjectList.jsx`)

**Current behavior:**
- Uses browser's native `confirm()` dialog (line 45)
- Blocks UI execution until user responds

**New behavior:**
- Custom HTML modal with confirmation message
- Consistent with existing modal styling (already used for "Create Project")
- Two buttons: "Cancel" and "Delete"

**Implementation:**
- Add new state: `deleteConfirm` to track which project is being deleted
- Add new state: `showDeleteModal` to control modal visibility
- Replace `confirm()` call with setting `showDeleteModal(true)` and `deleteConfirm(id)`
- Create new modal component similar to existing create modal (lines 99-135)
- Modal should include:
  - Warning message about deleting project and all posts
  - Cancel button (closes modal)
  - Delete button (calls `deleteProject()` and closes modal)
- Update `handleDelete` to work with modal instead of direct execution

## Files to Modify

1. `frontend/src/components/Sidebar.jsx`
   - Remove `projectsOpen` state
   - Replace collapsible Projects section with direct NavLink

2. `frontend/src/components/Projects/ProjectList.jsx`
   - Add `showDeleteModal` and `deleteConfirm` state
   - Update `handleDelete` to show modal instead of calling `confirm()`
   - Add delete confirmation modal JSX

## CSS Considerations

No CSS changes needed - existing modal styles (`.modal-overlay`, `.modal`, `.modal-footer`, etc.) will be reused for the delete confirmation modal.

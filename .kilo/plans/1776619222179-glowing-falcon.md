# Implementation Plan: Always-Visible Model Name Field

## Problem Statement

The NVIDIA NIM provider does not support model listing functionality. When this provider is selected, the Default Model Name input field becomes hidden because it only appears when `availableModels.length > 0`. This prevents users from manually entering model names for providers that don't support model enumeration.

## Current Behavior Analysis

**File:** `frontend/src/components/Settings/AIProviders.jsx`

1. **Model dropdown visibility** (lines 218-226):
   - Only renders when `availableModels.length > 0`
   - Hidden for providers that don't support model listing

2. **Fetch Models button** (lines 249-253):
   - Only shows for `openai_compatible` provider
   - Not available for OpenRouter or Nvidia NIM

3. **Model name storage**:
   - Stored in `form.model_name` state
   - Saved to database when provider is created/updated

## Solution Design

### Approach: Hybrid Input System

Always show a Model Name field that supports both:
- **Manual entry**: Text input for users to type model names
- **Dropdown selection**: When models are fetched, show a dropdown with options

### Implementation Strategy

#### 1. Always-Visible Model Name Field

**Location:** Lines 218-226 (replace current conditional rendering)

**New behavior:**
- Always show a Model Name input field
- Default to text input for manual entry
- When models are fetched, show a dropdown with fetched models
- Allow users to switch between dropdown and manual entry

#### 2. Enhanced Fetch Models Button

**Location:** Lines 249-253

**New behavior:**
- Show "Fetch Models" button for all OpenAI-compatible providers (openai_compatible, openrouter, nvidia_nim)
- For providers that don't support model listing (like Nvidia NIM), show a helpful message when fetch fails
- Keep the button visible so users can try fetching or see the error

#### 3. UI/UX Improvements

**Features:**
- Add a "Use custom model" option in dropdown to allow manual entry
- Show placeholder text indicating manual entry is always available
- Display helpful hint text for providers without model listing
- Maintain backward compatibility with existing providers

### Step 3: Add State for Custom Model Entry

**Location:** Line 13 (state initialization)

**Add new state:**
```jsx
const [useCustomModel, setUseCustomModel] = useState(false)
```

**Add handler for custom model toggle:**
```jsx
const handleModelChange = (value) => {
  if (value === '__custom__') {
    setUseCustomModel(true)
    setForm({ ...form, model_name: '' })
  } else {
    setUseCustomModel(false)
    setForm({ ...form, model_name: value })
  }
}
```

### Step 5: Reset State on Provider Change

**Location:** Lines 22-33 (useEffect)

**Add reset logic:**
```jsx
useEffect(() => {
  setForm(prevForm => {
    if (prevForm.provider_type === 'openrouter' && !prevForm.api_url) {
      return { ...prevForm, api_url: 'https://openrouter.ai/api/v1/' }
    } else if (prevForm.provider_type === 'nvidia_nim' && !prevForm.api_url) {
      return { ...prevForm, api_url: 'https://integrate.api.nvidia.com/v1/' }
    } else if (prevForm.provider_type !== 'openai_compatible' && prevForm.provider_type !== 'openrouter' && prevForm.provider_type !== 'nvidia_nim') {
      return { ...prevForm, api_url: '' }
    }
    return prevForm
  })
  setAvailableModels([])
  setUseCustomModel(false)
}, [form.provider_type])
```

# Plan: Post Detail UI Improvements

## Overview
Redesign the post detail page to improve UX with larger pipeline steps, automatic pipeline execution, better token tracking, and reorganized layout.

## Changes Required

### 1. Backend Changes

#### 1.1 Update TokenUsage Model (`backend/app/models/post.py`)
- Add `input_tokens` and `output_tokens` fields to track token usage separately
- Keep existing per-stage fields for backward compatibility

```python
class TokenUsage(BaseModel):
    research: int = 0
    outline: int = 0
    content: int = 0
    thumbnail: int = 0
    section_images: int = 0
    total: int = 0
    input_tokens: int = 0  # NEW
    output_tokens: int = 0  # NEW
```

#### 1.2 Update AI Service Functions (`backend/app/services/ai_service.py`)
- Modify `_call_openai`, `_call_gemini`, `_call_anthropic` to return `(text, input_tokens, output_tokens)` tuple
- Update all service functions to return separate input/output token counts

#### 1.3 Update Worker Tasks (`worker/app/workers/tasks.py`)
- Update `run_research`, `run_outline`, `run_content` to track and store `input_tokens` and `output_tokens`
- Update total token calculation to include both input and output

### 2. Frontend Changes

#### 2.1 Update PostView Component (`frontend/src/components/Posts/PostView.jsx`)

**Layout Reorganization:**
- Move action buttons (Generate Thumbnail, Generate Section Images, Publish/Unpublish) to be inline with post name and status
- Move pipeline progress section to appear after token usage section
- Remove "Generate Outline" and "Generate Content" buttons (they run automatically)

**New Structure:**
```
[Back Button]
[Page Header]
  - Post Title
  - Status Badge
  - Action Buttons (inline)
[Token Usage Section]
[Pipeline Progress Section] (larger, connected steps)
[Research Data Section]
[Outline Section]
[Content Preview Section]
[Job History Section]
```

## Implementation Steps

### Phase 1: Backend Token Tracking
1. Update `TokenUsage` model in `backend/app/models/post.py`
2. Modify AI service functions to return separate input/output tokens
3. Update worker tasks to track and store input/output tokens
4. Test token tracking with a sample post

### Phase 2: Frontend Layout Reorganization
1. Update PostView component structure
2. Move action buttons to inline with header
3. Remove manual generate buttons for outline/content
4. Reorder sections (token usage before pipeline progress)

### Phase 3: Enhanced Pipeline Visualization
1. Add new CSS styles for larger pipeline steps
2. Implement connected step visualization with lines
3. Add visual indicators for running/active stages
4. Test pipeline visualization with different states

### Phase 4: Testing & Refinement
1. Test full workflow from post creation to publishing
2. Verify token tracking accuracy
3. Test responsive behavior on different screen sizes
4. Polish animations and transitions

## Notes

- The automatic pipeline execution (research → outline → content) already exists in the backend
- Only thumbnail and section images require manual user action
- Input/output token tracking requires changes to all AI provider calls
- The pipeline visualization will use CSS pseudo-elements for connectors
- Consider adding a "Regenerate" option for failed stages (future enhancement)

# Plan: Create Two Admin Users on Backend Startup

## Objective
Modify backend startup to create two separate admin users:
1. Admin user with username "admin" and password from `ADMIN_PASSWORD`
2. Admin user with username from `INIT_USER` and password from `INIT_PASSWORD`

## Current State
- `backend/app/config.py` has `ADMIN_PASSWORD`, `INIT_USER`, and `INIT_PASSWORD` environment variables
- `backend/app/services/user_service.py` creates ONE admin account using `INIT_USER` and `INIT_PASSWORD`
- There's a check preventing `INIT_USER` from being "admin" (lines 17-23)
- `.env` file contains `ADMIN_PASSWORD=admin123`, `INIT_USER=tam`, `INIT_PASSWORD=tam123`

## Changes Required

### 1. Update `backend/app/config.py`
- Add `ADMIN_USER` environment variable with default value "admin"
- This makes the admin username configurable while defaulting to "admin"

### 2. Update `backend/app/services/user_service.py`
- Remove the check that prevents `INIT_USER` from being "admin" (lines 17-23)
- Modify `create_admin_account()` function to create TWO admin accounts:
  - First: username from `ADMIN_USER` (default "admin") with password from `ADMIN_PASSWORD`
  - Second: username from `INIT_USER` with password from `INIT_PASSWORD`
- Update logging to reflect creation of both accounts
- Handle the case where either account already exists (skip creation if exists)

### 3. Update `.env` file (if needed)
- Ensure `ADMIN_PASSWORD` is set (already present: `ADMIN_PASSWORD=admin123`)
- Optionally add `ADMIN_USER=admin` for clarity (though it will default to "admin")

## Implementation Details

### config.py Changes
```python
ADMIN_USER: str = os.getenv("ADMIN_USER", "admin")
ADMIN_PASSWORD: str = os.getenv("ADMIN_PASSWORD", "admin123")
```

### user_service.py Changes
The `create_admin_account()` function will:
1. Create admin account with `ADMIN_USER`/`ADMIN_PASSWORD` if it doesn't exist
2. Create admin account with `INIT_USER`/`INIT_PASSWORD` if it doesn't exist
3. Log success/failure for each account creation
4. Handle edge cases:
   - If `ADMIN_USER` == `INIT_USER`, only create one account
   - If either account already exists, skip creation

## Edge Cases to Handle
1. **Same username**: If `ADMIN_USER` equals `INIT_USER`, only create one account
2. **Existing accounts**: Skip creation if either account already exists
3. **Missing credentials**: Config validation already ensures `INIT_USER` and `INIT_PASSWORD` are present; `ADMIN_PASSWORD` has a default

## Testing Strategy
1. Start backend with fresh database
2. Verify two admin accounts are created
3. Verify both can login successfully
4. Restart backend (accounts already exist)
5. Verify no duplicate accounts are created
6. Test with `ADMIN_USER == INIT_USER` (should only create one account)

## Files to Modify
1. `backend/app/config.py` - Add `ADMIN_USER` setting
2. `backend/app/services/user_service.py` - Update `create_admin_account()` function
3. `.env` - Optionally add `ADMIN_USER=admin` for clarity (not required)

- `backend/app/database.py` - No changes needed

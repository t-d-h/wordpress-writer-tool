# Plan: Fix Model Name Input to Always Use Dropdown for AI Generation

## Overview
When users select AI generation for content, thumbnails, or section images, the Model Name field should always fetch available models from the provider API and display them as a dropdown. Manual text input should not be allowed.

## Problem
Current implementation falls back to a text input if:
1. Model fetching fails
2. No models are returned from the API
3. The provider is not OpenAI-compatible

This creates inconsistency and potential errors when users manually type model names.

## Solution
Remove the fallback to text input. Always show a dropdown with available models. If no models are available, show an error message or disable the option.

## Changes Required

### 1. Frontend Component Updates (`frontend/src/components/Projects/ProjectDetail.jsx`)

#### 1.1 Update `fetchModelsForProvider` function
- Add error handling to show user-friendly messages
- Set `availableModels` to empty array on error
- Keep `loadingModels` state for UI feedback

#### 1.2 Remove text input fallback in all three sections
- **Content AI Provider section**: Remove text input, only show dropdown
- **Thumbnail Options section**: Remove text input, only show dropdown
- **Section Images Options section**: Remove text input, only show dropdown

#### 1.3 Add error state and messaging
- Add `modelError` state to track fetch errors
- Show error message when model fetching fails
- Disable "Create" button if required model fields are empty

#### 1.4 Update model selection logic
- When provider changes, clear model selection
- When models load successfully, auto-select first model if none selected
- Show "Loading models..." while fetching
- Show "No models available" if fetch returns empty array

### 2. Implementation Details

#### For each AI provider selection (Content, Thumbnail, Section Images):

**Current behavior:**
```jsx
{loadingModels ? (
  <div>Loading models...</div>
) : availableModels.length > 0 ? (
  <select>...</select>
) : (
  <input placeholder="e.g. llama-3.3-70b-versatile" />
)}
```

**New behavior:**
```jsx
{loadingModels ? (
  <div>Loading models...</div>
) : availableModels.length > 0 ? (
  <select>...</select>
) : (
  <div style={{ color: 'var(--error)', fontSize: 14 }}>
    No models available for this provider
  </div>
)}
```

#### Add validation before form submission:
- Check that if `thumbnail_source === 'ai'`, then `thumbnail_provider_id` and `thumbnail_model_name` are set
- Check that if `section_images_source === 'ai'`, then `section_images_provider_id` and `section_images_model_name` are set
- Show error alert if validation fails

- [ ] Select Gemini provider for content → no model dropdown shown (Gemini uses fixed model)
- [ ] Select OpenAI provider for content → no model dropdown shown (OpenAI uses fixed model)
- [ ] Select AI generation for thumbnail → models load and show in dropdown
- [ ] Select AI generation for section images → models load and show in dropdown
- [ ] Try to create post without selecting model → validation error shown
- [ ] Model fetch fails → error message shown, no text input fallback

- [ ] Bulk creation with AI generation → same behavior as single post

### 4. Edge Cases & Considerations

- **Non-OpenAI-compatible providers**: Gemini and OpenAI use fixed models, so no dropdown should appear
- **Model fetch failure**: Show error message, don't allow form submission
- **Empty model list**: Show "No models available" message
- **Provider without API key**: Fetch will fail, show error message
- **Network issues**: Show error message, allow retry by selecting provider again

### 5. UI/UX Improvements

- Add loading spinner or progress indicator while fetching models
- Show error message in red when model fetch fails
- Add tooltip explaining that models are fetched from provider API
- Disable "Create" button when required model fields are empty
- Show model count in dropdown (e.g., "Select model (5 available)")

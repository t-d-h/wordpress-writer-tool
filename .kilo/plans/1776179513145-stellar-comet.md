# Plan: Replace All Posts Tab with Table View

## Milestone Summary

**Milestone v1.1: All Posts Table View**

**Goal:** Replace the "All Posts" tab in ProjectDetail with a table view similar to /all-posts, keeping search/sort/filter functionality while using manual pagination.

**Target features:**
- Replace PostCard grid layout with table layout
- Use /all-posts columns: Title, URL, Categories, Tags, Date, Status, Actions
- Keep search by title functionality
- Keep sort by date/title/status functionality
- Keep status filter functionality
- Switch from infinite scroll to manual pagination (100 posts per page)
- Scope to project's WordPress site only (no site selection dropdown)
- Discard PostCard component usage in this tab
- Discard origin badges (not needed in table view)

**Key context:**
- Current All Posts tab uses PostCard grid with infinite scroll
- Target /all-posts page uses table with manual pagination
- User wants table view but with enhanced filtering (search + sort + status filter)
- ProjectDetail already has the WordPress site context, so no site selection needed

## Research Complete

**Stack additions:** None (only httpx needs to be added to requirements.txt)
**Feature table stakes:** Table layout with Title, URL, Categories, Tags, Date, Status, Actions columns
**Watch Out For:**
- Incomplete state migration from infinite scroll to manual pagination
- Breaking search/sort/filter state reset logic
- Forgetting to remove PostCard component and related styling
- Incorrect API parameter mapping for 100 posts per page
- WordPress REST API nested data transformation (_embedded['wp:term'])

## Requirements Definition

Based on research, the requirements fall into these categories:

### Backend API
- Expose search, orderby, and order parameters in get_site_posts endpoint
- Support 100 posts per page pagination
- Return WordPress REST API data with proper transformation

### Frontend UI
- Table layout with columns: Title, URL, Categories, Tags, Date, Status, Actions
- Search by title input field
- Sort dropdown (date, title, status)
- Status filter dropdown
- Manual pagination controls (Previous/Next buttons)
- Loading states for pagination changes
- Empty state handling

### Cleanup
- Remove PostCard component usage from All Posts tab
- Remove origin badges
- Remove infinite scroll logic and event listeners
- Remove unused state variables

### Data Transformation
- Transform WordPress REST API response to table format
- Handle nested categories and tags from _embedded['wp:term']
- Format dates for display
- Generate edit URLs for WordPress admin

### Files to Modify
- `frontend/src/components/Login.jsx` (line 71)

- `frontend/src/components/Settings/WPSites.jsx` (line 171)

### Implementation Approach

**Option A: Create Reusable PasswordInput Component** (Recommended)
- Create `frontend/src/components/PasswordInput.jsx`
- Component props: `id`, `value`, `onChange`, `placeholder`, `required`, `disabled`, `className`
- Internal state: `showPassword` (boolean)
- Toggle between `type="password"` and `type="text"`
- Use `HiOutlineEye` and `HiOutlineEyeSlash` icons from `react-icons/hi2`
- Style: Position eye icon absolutely on the right side of input

**Option B: Add Eye Icon to Each Field Individually**
- Add state to each component for each password field
- Add eye icon button next to each password input
- More code duplication but no new component

### Recommended: Option A (Reusable Component)

**PasswordInput Component Structure:**
```jsx
import { useState } from 'react'
import { HiOutlineEye, HiOutlineEyeSlash } from 'react-icons/hi2'

export default function PasswordInput({ id, value, onChange, placeholder, required, disabled, className }) {
  const [showPassword, setShowPassword] = useState(false)

  return (
    <div style={{ position: 'relative' }}>
      <input
        id={id}
        className={className}
        type={showPassword ? 'text' : 'password'}
        value={value}
        onChange={onChange}
        placeholder={placeholder}
        required={required}
        disabled={disabled}
        style={{ paddingRight: '40px' }}
      />
      <button
        type="button"
        onClick={() => setShowPassword(!showPassword)}
        style={{
          position: 'absolute',
          right: '10px',
          top: '50%',
          transform: 'translateY(-50%)',
          background: 'none',
          border: 'none',
          cursor: 'pointer',
          color: 'var(--text-muted)',
          display: 'flex',
          alignItems: 'center'
        }}
      >
        {showPassword ? <HiOutlineEyeSlash size={18} /> : <HiOutlineEye size={18} />}
      </button>
    </div>
  )
}
```

**Changes to Existing Files:**
1. Replace password `<input>` elements with `<PasswordInput>` component
2. Import the component at the top of each file
3. Pass appropriate props

## Task 4: Fix Duplicate Username in Sidebar

### Investigation Required

**Current State:**
- Username is displayed once in `frontend/src/components/Sidebar.jsx` at line 155
- Code shows: `<span className="user-name">{user?.username || 'Guest'}</span>`
- No other instances of username display found in codebase

**Possible Causes:**
1. **CSS duplication**: Styles might be causing visual duplication
2. **React rendering issue**: Component might be rendering twice
3. **Browser rendering issue**: Display artifact
4. **User perception**: Username might appear in multiple contexts (e.g., page title, browser tab)

**Investigation Steps:**
1. Run the application and visually inspect the sidebar footer
2. Check browser DevTools for duplicate elements in the DOM
3. Verify if the username and role spans are overlapping or rendering incorrectly
4. Check if there's a CSS display issue causing visual duplication
5. Verify the Sidebar component isn't mounted twice (React StrictMode)

**Potential Fixes:**
- **Fix 1: Add proper spacing between username and role**
  - Add `display: 'block'` or `display: 'flex'` with gap to user-info div
  - Ensure username and role are on separate lines or properly spaced
- **Fix 2: Check CSS for overlapping elements**
  - Add explicit positioning to prevent overlap
  - Ensure proper z-index if elements are stacking
- **Fix 3: Verify component mounting**
  - Check if Sidebar is rendered multiple times in Layout
  - Ensure no duplicate component instances

**Current Code Analysis:**
```jsx
<div className="user-info" style={{ marginBottom: '12px' }}>
  <span className="user-name">{user?.username || 'Guest'}</span>
  <span className="user-role">{user?.role || ''}</span>
</div>
```

Both spans are inline elements without explicit spacing, which could cause them to render on top of each other or appear duplicated if there's a CSS issue.

**Recommended Fix:**
Update the user-info div to use flexbox with proper spacing:
```jsx
<div className="user-info" style={{ marginBottom: '12px', display: 'flex', flexDirection: 'column', gap: '4px' }}>
  <span className="user-name">{user?.username || 'Guest'}</span>
  <span className="user-role">{user?.role || ''}</span>
</div>
```

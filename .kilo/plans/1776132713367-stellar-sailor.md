## Why `/gsd-new-project` is Not the Right Command

The `/gsd-new-project` command is for initializing a brand new project from scratch. It creates:
- `.planning/PROJECT.md` - project context
- `.planning/config.json` - workflow preferences
- `.planning/research/` - domain research
- `.planning/REQUIREMENTS.md` - scoped requirements
- `.planning/ROADMAP.md` - phase structure
- `.planning/STATE.md` - project memory

Since this project already exists with all these artifacts, running `/gsd-new-project` would overwrite or conflict with existing planning documents.

1. Update ProjectDetail.jsx to display token usage statistics

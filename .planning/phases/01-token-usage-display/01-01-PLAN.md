**Phase:** 01 - Token Usage Display
**Wave:** 1
**Status:** Not started

**type:** implementation

## Objective

Implement backend foundation for token usage aggregation including database indexes and API endpoint modifications.

## Requirements

- TOKEN-03: System shows total output tokens across all post types
- TOKEN-05: System calculates token usage on-the-fly from posts collection
- PERF-01: Token usage aggregation completes within 1 second for projects with <100 posts
- PERF-03: System implements database indexes for token usage queries
- DATA-01: System maintains accurate token usage counts across all post types

## must_haves

### truths
- User can view token usage breakdown in Project general tab above statistics section
- System displays token usage breakdown by post type (research, outline, content, thumbnail)
- System shows total input tokens and total output tokens across all post types
- System includes deleted posts in token usage calculations
- Token usage display loads within 1 second for projects with <100 posts

### artifacts
- `backend/app/database.py` - Database indexes for token usage queries
- `backend/app/routers/projects.py` - Token usage aggregation endpoint
- `backend/app/models/project.py` - Updated response model with token usage fields

### key_links
- Database indexes → MongoDB aggregation pipeline performance
- Aggregation endpoint → Frontend token usage display
- Response model → API client data structure

    Add categories and tags fetching to WordPress sync service:
    1. Open backend/app/services/wp_service.py
    2. Find the function that syncs posts from WordPress (search for functions that fetch posts)
    3. When fetching a post from WordPress REST API, also fetch its categories and tags
    4. WordPress REST API endpoints:
       - Categories: GET /wp/v2/categories?post={post_id}
       - Tags: GET /wp/v2/tags?post={post_id}
    5. Extract category/tag names from the API responses
    6. Store categories and tags in the post document in MongoDB

    Implementation approach:
    - After fetching post data from /wp/v2/posts/{post_id}, make two additional API calls
    - For categories: GET {wp_site_url}/wp/v2/categories?post={post_id}
    - For tags: GET {wp_site_url}/wp/v2/tags?post={post_id}
    - Extract the 'name' field from each category/tag object
    - Store as lists of strings in the post document

    Example code to add in the post sync function:
    ```python
    # Fetch categories
    categories_response = await client.get(
        f"{wp_site_url}/wp/v2/categories",
        params={"post": wp_post_id}
    )
    categories = [cat["name"] for cat in categories_response.json()]

    # Fetch tags
    tags_response = await client.get(
        f"{wp_site_url}/wp/v2/tags",
        params={"post": wp_post_id}
    )
    tags = [tag["name"] for tag in tags_response.json()]

    # Store in post document
    post_doc["categories"] = categories
    post_doc["tags"] = tags
    ```

    This follows the existing pattern in wp_service.py for fetching WordPress data using httpx.
  </action>
  <verify>
    <automated>grep -n "wp/v2/categories" backend/app/services/wp_service.py && grep -n "wp/v2/tags" backend/app/services/wp_service.py</automated>
  </verify>
  <done>WordPress sync service fetches categories and tags and stores them in MongoDB</done>

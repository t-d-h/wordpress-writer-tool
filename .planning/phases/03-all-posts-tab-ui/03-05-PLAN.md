# Plan 05: Testing and Verification

## Objective
Test the All Posts tab functionality to ensure all requirements are met. This includes manual testing of the UI, API endpoints, and integration with WordPress sync.

## Requirements
All requirements from POSTS, PERF, DATA, and UX categories for Phase 3

## Context from 03-CONTEXT.md
- All implementation decisions from Plans 01-04
- All requirements from REQUIREMENTS.md

## Tasks

### Task 05-01: Manual UI testing
**File**: Manual testing checklist

**Steps**:
1. Navigate to a project in the UI
2. Click on "All Posts" tab
3. Verify tab appears and is active
4. Verify posts are displayed in card layout
5. Verify each card shows: title, status, date, origin badge, edit button
6. Verify origin badge shows "Tool" or "Existing" correctly
7. Click edit button and verify WordPress admin opens in new tab
8. Verify empty state shows when no posts exist
9. Verify loading spinner shows while fetching posts
10. Verify error message shows if fetch fails

**Acceptance Criteria**:
- [ ] "All Posts" tab appears and is active
- [ ] Posts are displayed in card layout
- [ ] Each card shows title, status, date, origin badge, edit button
- [ ] Origin badge shows "Tool" or "Existing" correctly
- [ ] Edit button opens WordPress admin in new tab
- [ ] Empty state shows when no posts exist
- [ ] Loading spinner shows while fetching posts
- [ ] Error message shows if fetch fails

### Task 05-02: Manual filter testing
**File**: Manual testing checklist

**Steps**:
1. Navigate to All Posts tab
2. Select different status filters (all, published, draft, pending, failed)
3. Verify posts are filtered correctly
4. Verify empty state shows if no posts match filter
5. Verify filter works in combination with sort and search

**Acceptance Criteria**:
- [ ] Status filter dropdown shows all options
- [ ] Posts are filtered correctly by status
- [ ] Empty state shows if no posts match filter
- [ ] Filter works with sort and search

### Task 05-03: Manual sort testing

3. Verify posts are sorted correctly
4. Verify sort works in combination with filter and search

**Acceptance Criteria**:

### Task 05-04: Manual search testing
**File**: Manual testing checklist

**Steps**:

3. Verify posts are filtered by title (case-insensitive)
4. Verify empty state shows if no posts match search

**Acceptance Criteria**:

- [ ] Posts are filtered by title (case-insensitive)
- [ ] Empty state shows if no posts match search
- [ ] Search works with filter and sort
- [ ] Search updates in real-time

### Task 05-05: Manual infinite scroll testing
**File**: Manual testing checklist

**Steps**:
1. Navigate to All Posts tab with many posts (>20)
2. Scroll down to bottom of page
3. Verify more posts load automatically
4. Verify loading indicator shows while loading more posts
5. Verify loading stops when no more posts available
6. Change filter/sort/search and verify pagination resets
7. Verify posts are reloaded with new parameters

**Acceptance Criteria**:
- [ ] More posts load when scrolling near bottom
- [ ] Loading indicator shows while loading more posts
- [ ] Loading stops when no more posts available
- [ ] Pagination resets when filter/sort/search changes
- [ ] Posts are reloaded with new parameters

### Task 05-06: API endpoint testing
**File**: Manual testing checklist

**Steps**:
1. Test GET /api/projects/{project_id}/posts endpoint
2. Test with different query parameters (page, limit, status, sort_by, search)
3. Verify response includes posts array and total count
4. Verify pagination works correctly
5. Verify filter works correctly
6. Verify sort works correctly
7. Verify search works correctly
8. Verify errors are handled correctly

**Acceptance Criteria**:
- [ ] API endpoint returns posts array and total count
- [ ] Pagination works correctly (page, limit)
- [ ] Filter works correctly (status)
- [ ] Sort works correctly (sort_by)
- [ ] Search works correctly (search)
- [ ] Errors are handled correctly

### Task 05-07: Performance testing
**File**: Manual testing checklist

**Steps**:
1. Test with project with <200 posts
2. Measure load time for All Posts tab
3. Verify load time is <2 seconds (PERF-02)
4. Test infinite scroll performance
5. Verify no noticeable lag when scrolling

**Acceptance Criteria**:
- [ ] All Posts tab loads within 2 seconds for projects with <200 posts
- [ ] Infinite scroll is smooth with no noticeable lag
- [ ] Filter/sort/search operations are fast

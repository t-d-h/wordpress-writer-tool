# Plan 02: Implement Filter, Sort, and Search Functionality

## Objective
Add control bar with status filter, sort dropdown, and search input to the All Posts tab. These controls will filter and sort the displayed posts.

## Requirements
- POSTS-10: User can filter posts by status (published, draft, pending, etc.)
- POSTS-11: User can sort posts by date
- POSTS-12: User can search posts by title
- UX-04: Filter controls are intuitive and responsive
- UX-05: Search functionality provides real-time feedback

## Context from 03-CONTEXT.md
- **D-07**: Status filter placed at top of tab, above post list
- **D-08**: Filter dropdown shows status options (all, published, draft, pending, etc.)
- **D-09**: Filter dropdown positioned in control bar at top
- **D-10**: Sort dropdown placed at top of tab, next to filter
- **D-11**: Sort options include date (newest/oldest), title (A-Z/Z-A), status
- **D-12**: Sort dropdown positioned in control bar at top
- **D-13**: Search input placed at top of tab, next to filter and sort
- **D-14**: Search filters by post title
- **D-15**: Search input positioned in control bar at top

**File**: `frontend/src/components/Projects/ProjectDetail.jsx`

**Steps**:

### Task 02-02: Implement filter logic
**File**: `frontend/src/components/Projects/ProjectDetail.jsx`

**Steps**:
1. Create filteredPosts array by filtering allPosts based on statusFilter
2. If statusFilter is 'all', show all posts
3. If statusFilter is specific status, show only posts with that status
4. Update PostCard rendering to use filteredPosts instead of allPosts
5. Add useEffect to re-filter when statusFilter changes

**Acceptance Criteria**:
- [ ] Filter dropdown filters posts by status
- [ ] "all" option shows all posts
- [ ] Specific status option shows only posts with that status
- [ ] Filter updates immediately when selection changes
- [ ] Empty state shows if no posts match filter

### Task 02-03: Implement sort logic
**File**: `frontend/src/components/Projects/ProjectDetail.jsx`

**Steps**:
1. Create sortedPosts array by sorting filteredPosts based on sortBy
2. Implement sort options:
   - date-desc: newest first (created_at or updated_at)
   - date-asc: oldest first
   - title-asc: A-Z
   - title-desc: Z-A
   - status: alphabetical by status
3. Update PostCard rendering to use sortedPosts instead of filteredPosts
4. Add useEffect to re-sort when sortBy changes

**Acceptance Criteria**:
- [ ] Sort dropdown sorts posts by selected criteria
- [ ] date-desc shows newest posts first
- [ ] date-asc shows oldest posts first
- [ ] title-asc shows posts in A-Z order
- [ ] title-desc shows posts in Z-A order
- [ ] status sorts posts alphabetically by status
- [ ] Sort updates immediately when selection changes

### Task 02-04: Implement search logic
**File**: `frontend/src/components/Projects/ProjectDetail.jsx`

**Steps**:
1. Create searchedPosts array by filtering sortedPosts based on searchQuery
2. Filter posts where title includes searchQuery (case-insensitive)
3. Update PostCard rendering to use searchedPosts instead of sortedPosts
4. Add useEffect to re-search when searchQuery changes
5. Debounce search input to avoid excessive filtering (optional, for performance)

**Acceptance Criteria**:
- [ ] Search input filters posts by title
- [ ] Search is case-insensitive
- [ ] Search updates as user types (real-time feedback)
- [ ] Empty search shows all posts
- [ ] Empty state shows if no posts match search
- [ ] Search works in combination with filter and sort

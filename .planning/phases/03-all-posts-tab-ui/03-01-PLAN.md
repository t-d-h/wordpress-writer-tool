# Plan 01: Add All Posts Tab to ProjectDetail

## Objective
Add "All Posts" tab to ProjectDetail component as the third tab (after general and content). This tab will display all WordPress posts for the project with filtering, sorting, and search capabilities.

## Requirements
- POSTS-01: User can view "All Posts" tab in each project
- POSTS-02: System displays all WordPress posts (both tool-created and existing)
- POSTS-03: System provides visual distinction between tool-created and existing posts
- POSTS-04: User can click Edit button to open WordPress admin edit page in new tab
- POSTS-05: System displays post name/title
- POSTS-06: System displays post URL
- POSTS-07: System displays post categories
- POSTS-08: System displays post tags
- POSTS-09: System displays post date
- UX-02: All Posts tab provides clear visual indicators for post types
- UX-03: Edit button is clearly visible and accessible

## Context from 03-CONTEXT.md
- **D-01**: Use card-based layout for post display (reuses existing Card component)
- **D-02**: Card shows essential information only: title, status, date, origin badge, edit button
- **D-03**: Cards arranged in grid layout (consistent with existing stat-card pattern)
- **D-04**: Use badge to distinguish tool-created vs existing posts
- **D-05**: Badge shows "Tool" for tool-created posts, "Existing" for WordPress posts
- **D-06**: Badge placed on card (position at agent's discretion)

### Task 01-02: Create PostCard component
**File**: `frontend/src/components/Projects/PostCard.jsx` (new file)

**Steps**:
1. Create new PostCard component
2. Accept props: post (object), onEdit (function)
3. Render card with essential information:
   - Title (post.title)
   - Status (post.status)
   - Date (formatted post.created_at or post.updated_at)
   - Origin badge (post.origin === 'tool' ? 'Tool' : 'Existing')
   - Edit button (opens WordPress admin edit page)
4. Use stat-card CSS class for styling
5. Handle loading and error states

**Acceptance Criteria**:
- [ ] PostCard component renders card with title, status, date, origin badge, edit button
- [ ] Badge shows "Tool" for tool-created posts, "Existing" for WordPress posts
- [ ] Edit button opens WordPress admin edit page in new tab
- [ ] Card uses stat-card CSS class
- [ ] Component handles null/undefined post gracefully

### Task 01-03: Integrate PostCard into All Posts tab
**File**: `frontend/src/components/Projects/ProjectDetail.jsx`

**Steps**:
1. Import PostCard component
2. Add state for posts list: `const [allPosts, setAllPosts] = useState([])`
3. Add loading state: `const [loadingAllPosts, setLoadingAllPosts] = useState(false)`
4. Add error state: `const [allPostsError, setAllPostsError] = useState(null)`

6. Call loadAllPosts() in useEffect when activeTab === 'all-posts'
7. Render PostCard components in grid layout

**Acceptance Criteria**:
- [ ] All Posts tab displays list of PostCard components

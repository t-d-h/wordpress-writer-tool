# Plan 04: Backend API Integration for All Posts

## Objective
Extend backend API to support fetching all posts for a project with filter, sort, and search parameters. This API will be used by the frontend All Posts tab.

## Requirements
- POSTS-13: Backend provides WordPress API method to fetch all posts
- POSTS-14: System tracks post origin in database (tool-created vs existing)
- PERF-02: All Posts tab loads within 2 seconds for projects with <200 posts
- DATA-02: System correctly identifies post origin (tool-created vs existing)

## Context from 03-CONTEXT.md
- Backend WordPress sync endpoints from Phase 02 are available
- Post model already has `origin` field from Phase 02
- WordPress service with search/filtering is available from Phase 02

## Tasks

### Task 04-01: Add get_all_posts endpoint to projects router
**File**: `backend/app/routers/projects.py`

**Steps**:
1. Add new endpoint: GET /api/projects/{project_id}/posts
2. Accept query parameters:
   - page (int, default 1)
   - limit (int, default 20)
   - status (str, optional - filter by status)
   - sort_by (str, default 'date-desc')
   - search (str, optional - search by title)
3. Query posts collection for project
4. Apply status filter if provided
5. Apply search filter if provided (title contains search string, case-insensitive)
6. Apply sort based on sort_by parameter
7. Apply pagination (skip = (page - 1) * limit, limit = limit)
8. Return paginated results with total count
9. Handle errors with HTTPException

**Acceptance Criteria**:
- [ ] GET /api/projects/{project_id}/posts endpoint exists
- [ ] Endpoint accepts page, limit, status, sort_by, search query parameters
- [ ] Endpoint returns paginated posts with total count
- [ ] Status filter works correctly
- [ ] Search filter works correctly (case-insensitive)
- [ ] Sort works correctly (date-desc, date-asc, title-asc, title-desc, status)
- [ ] Pagination works correctly (skip and limit)
- [ ] Errors are handled with HTTPException

### Task 04-02: Add get_all_posts function to API client

**Steps**:

2. Accept parameters: projectId, page, limit, status, sortBy, search

4. Return response with posts array and total count

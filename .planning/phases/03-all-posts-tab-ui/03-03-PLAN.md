# Plan 03: Implement Infinite Scroll Pagination

## Objective
Implement infinite scroll to load more posts as user scrolls down. This provides a seamless experience without manual pagination controls.

## Requirements
- POSTS-02: System displays all WordPress posts (both tool-created and existing)
- PERF-02: All Posts tab loads within 2 seconds for projects with <200 posts
- UX-04: Filter controls are intuitive and responsive

## Context from 03-CONTEXT.md
- **D-18**: Use infinite scroll to load more posts as user scrolls down
- **D-19**: Seamless experience without manual pagination controls
- **D-23**: Loading state for infinite scroll (spinner, skeleton, etc.) - at agent's discretion

## Tasks

### Task 03-01: Add infinite scroll state and logic
**File**: `frontend/src/components/Projects/ProjectDetail.jsx`

**Steps**:
1. Add state for pagination:
   - `const [page, setPage] = useState(1)`
   - `const [hasMore, setHasMore] = useState(true)`
   - `const [loadingMore, setLoadingMore] = useState(false)`
2. Create loadMorePosts() function to fetch next page of posts
3. Append new posts to existing allPosts array (don't replace)
4. Set hasMore to false if no more posts available
5. Add useEffect to load more posts when page changes

**Acceptance Criteria**:
- [ ] Pagination state is initialized (page, hasMore, loadingMore)
- [ ] loadMorePosts() function fetches next page of posts
- [ ] New posts are appended to existing posts array
- [ ] hasMore is set to false when no more posts available
- [ ] Loading state is managed correctly

### Task 03-02: Implement scroll detection
**File**: `frontend/src/components/Projects/ProjectDetail.jsx`

**Steps**:

**Acceptance Criteria**:
- [ ] Scroll event listener is attached

- [ ] loadMorePosts() is called when near bottom and hasMore is true

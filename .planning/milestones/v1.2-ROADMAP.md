# Roadmap: v1.2 Vietnamese Language Support

**Milestone**: v1.2
**Status**: Planned
**Last Updated**: 2026-04-15

## Overview

This roadmap defines the phased execution plan for adding Vietnamese language support to the WordPress Writer Tool. The implementation is divided into 3 phases, each building on the previous one.

## Phase 12: Backend Foundation

**Goal**: Establish backend data model and validation for language support

**Requirements**:
- LANG-03: Language Field in Post Model
- LANG-08: Language Validation
- LANG-09: Backward Compatibility

**Success Criteria**:
- [ ] `language` field added to `PostCreate` and `BulkPostCreate` Pydantic models
- [ ] `language` field stored in MongoDB posts collection
- [ ] API validates language is "vietnamese" or "english"
- [ ] Invalid language returns 400 error with clear message
- [ ] API responses default to "english" for posts without language field
- [ ] No errors when querying or updating posts without language field
- [ ] All backend tests pass

**Dependencies**:
- None (can start immediately)

**Estimated Duration**: 2-3 hours

## Phase 13: AI Service Integration

**Goal**: Integrate language parameter into AI content generation pipeline

**Requirements**:
- LANG-04: Language Parameter in AI Service
- LANG-05: Language-Specific System Prompts
- LANG-07: Language in Job Payload

**Success Criteria**:
- [ ] `research_topic()` accepts `language` parameter
- [ ] `generate_outline()` accepts `language` parameter
- [ ] `generate_section_content()` accepts `language` parameter
- [ ] `generate_introduction()` accepts `language` parameter
- [ ] Language parameter passed from job payload to AI service functions
- [ ] Language parameter logged at each stage for debugging
- [ ] System prompts include explicit language instruction
- [ ] Vietnamese prompt includes cultural context and formality level
- [ ] All AI providers use language-specific prompts
- [ ] `language` field included in all job payloads (research, outline, content, thumbnail, section_images, publish)
- [ ] Language field validated in job payload
- [ ] End-to-end test: Vietnamese post generates Vietnamese content
- [ ] End-to-end test: English post generates English content

**Dependencies**:
- Phase 12: Backend Foundation (must complete first)

**Estimated Duration**: 3-4 hours

## Phase 14: Frontend UI

**Goal**: Add user interface for language selection and display

**Requirements**:
- LANG-01: Language Selection UI
- LANG-02: Language Display in Post List
- LANG-06: Language Persistence

**Success Criteria**:
- [ ] Radio buttons displayed after "Topic" field, before "Additional Requests"
- [ ] Two options: "Tiếng Việt" (Vietnamese) and "English"
- [ ] Vietnamese is selected by default
- [ ] Visual indication of selected language (radio button state)
- [ ] Language badge shown in post list table
- [ ] Language badge shown in post detail view
- [ ] Badge uses color coding (green for Vietnamese, blue for English)
- [ ] Language selection persists when form is submitted with errors
- [ ] Language selection persists when user navigates away and returns
- [ ] Language selection stored in localStorage or form state
- [ ] Language selection resets to default (Vietnamese) on page refresh
- [ ] Frontend tests pass
- [ ] Manual testing: User can create Vietnamese post
- [ ] Manual testing: User can create English post
- [ ] Manual testing: Language badge displays correctly

**Dependencies**:
- Phase 12: Backend Foundation (must complete first)
- Phase 13: AI Service Integration (must complete first)

**Estimated Duration**: 2-3 hours

## Risk Mitigation

### Risk: Vietnamese content quality varies across AI providers
**Mitigation**: Test with all three providers (OpenAI, Gemini, Anthropic) and document quality differences. Consider provider-specific prompt tuning if needed.

### Risk: Language parameter not passed through pipeline correctly
**Mitigation**: Add logging at each stage to verify language parameter flow. Create end-to-end tests to validate entire pipeline.

### Risk: Backward compatibility issues with existing posts
**Mitigation**: Default to "english" for posts without language field. Test with existing posts to ensure no errors.

### Risk: Vietnamese cultural context not captured in prompts
**Mitigation**: Include cultural context in system prompts. Test with Vietnamese speakers for cultural appropriateness.

## Notes

- Vietnamese is a "low-resource" language with unique challenges (tonal structure, cultural context, limited training data)
- All existing AI providers (OpenAI GPT-4o, Gemini 2.0 Flash, Anthropic Claude Sonnet 4) support Vietnamese natively
- No new libraries required - implementation is configuration changes only
- Direct generation in target language (no translation workflow) is simpler and more cost-effective
- Test with Vietnamese speakers for cultural appropriateness

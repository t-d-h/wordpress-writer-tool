# Requirements: v1.3 Content Quality Improvements

**Milestone**: v1.3
**Status**: Shipped
**Last Updated**: 2026-04-17

## Overview

Fix content generation issues to improve output quality and ensure AI-generated content meets user specifications. This milestone focuses on HTML cleaning, validation (word count and section count), and pipeline integration (research data utilization).

## In-Scope Requirements

### HTML Cleaning

#### HTML-01: Clean HTML Output
**Priority**: High
**Description**: Remove unwanted characters (backticks, markdown artifacts) from AI-generated content
**Acceptance Criteria**:
- [x] System removes backticks from AI-generated content
- [x] System removes markdown code blocks wrapping HTML
- [x] System removes markdown artifacts from final content
- [x] Content is ready for WordPress without manual cleanup
- [x] HTML cleaning applied to all AI-generated content (research, outline, content)
**Notes**: Use BeautifulSoup4 + lxml for robust HTML sanitization

#### HTML-02: No Markdown Artifacts
**Priority**: High
**Description**: Ensure content doesn't contain markdown code blocks or backticks
**Acceptance Criteria**:
- [x] Final content doesn't contain markdown code blocks (```html or ```)
- [x] Final content doesn't contain backticks wrapping HTML
- [x] Content displays correctly in WordPress editor
- [x] No manual cleanup needed by users
**Notes**: AI often wraps HTML in markdown code blocks that need removal

#### HTML-03: HTML Sanitization
**Priority**: Medium
**Description**: Implement HTML sanitization with allowed tags whitelist
**Acceptance Criteria**:
- [x] System sanitizes HTML content before saving
- [x] Whitelist of allowed HTML tags defined (h1-h6, p, strong, em, ul, ol, li, a, img)
- [x] Malicious or unwanted tags removed
- [x] Content structure preserved
**Notes**: Define whitelist based on WordPress requirements

### Validation

#### VAL-01: Word Count Validation
**Priority**: High
**Description**: Validate actual word count against target word count
**Acceptance Criteria**:
- [x] System calculates actual word count using textstat
- [x] System compares actual word count to target word count
- [x] Validation uses ±20% tolerance
- [x] System warns user when outside tolerance
- [x] Validation results stored in post document
**Notes**: Use textstat for accurate word counting

#### VAL-02: Section Count Validation
**Priority**: High
**Description**: Validate actual section count against target section count
**Acceptance Criteria**:
- [x] System counts sections in outline structure
- [x] System compares actual section count to target section count
- [x] Validation uses ±1 section tolerance
- [x] System warns user when outside tolerance
- [x] Validation results stored in post document
**Notes**: Validate section count against outline structure (not content)

#### VAL-03: Validation Results Display
**Priority**: Medium
**Description**: Display validation results to users
**Acceptance Criteria**:
- [x] Word count validation results shown in post detail view
- [x] Section count validation results shown in post detail view
- [x] HTML cleanliness status shown in post detail view
- [x] Clear visual indication when validation fails
- [x] Users can see actual vs target values
**Notes**: Validation is advisory, not blocking

#### VAL-04: Validation Warnings
**Priority**: Medium
**Description**: Log warnings when content doesn't meet validation criteria
**Acceptance Criteria**:
- [x] System logs warning when word count validation fails
- [x] System logs warning when section count validation fails
- [x] System logs warning when HTML cleaning removes content
- [x] Warnings include actual vs target values
- [x] Warnings stored in job logs for debugging
**Notes**: Validation is non-blocking, content is still saved

#### VAL-05: Tolerance-Based Validation
**Priority**: Medium
**Description**: Implement tolerance-based validation instead of strict enforcement
**Acceptance Criteria**:
- [x] Word count validation uses ±20% tolerance
- [x] Section count validation uses ±1 section tolerance
- [x] Validation passes when within tolerance
- [x] Validation fails when outside tolerance
- [x] Users can publish content that doesn't meet validation
**Notes**: AI models cannot guarantee exact specifications

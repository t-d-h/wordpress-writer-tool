# Requirements: v1.3 Content Quality Improvements

**Milestone**: v1.3
**Status**: Shipped
**Last Updated**: 2026-04-18
**Phases Completed**: 15-16 (2 phases)
**Phases Deferred**: 17-21 (5 phases)

## Overview

Fix content generation issues to improve output quality and ensure AI-generated content meets user specifications. This milestone focuses on HTML cleaning and word count validation. Section count validation, validation results display, validation warnings, and research data utilization were deferred to v1.4.

## In-Scope Requirements (Completed)

### HTML Cleaning

#### HTML-01: Clean HTML Output
**Priority**: High
**Status**: ✅ Complete
**Outcome**: Validated
**Description**: Remove unwanted characters (backticks, markdown artifacts) from AI-generated content
**Acceptance Criteria**:
- [x] System removes backticks from AI-generated content
- [x] System removes markdown code blocks wrapping HTML
- [x] System removes markdown artifacts from final content
- [x] Content is ready for WordPress without manual cleanup
- [x] HTML cleaning applied to all AI-generated content (research, outline, content)
**Notes**: Used BeautifulSoup4 + lxml for robust HTML sanitization
**Evidence**:
- clean_html() function implemented with 5-stage algorithm
- Integration into generate_section_content() and generate_introduction()
- Test coverage: test_clean_html_strips_backticks, test_clean_html_strips_code_blocks

#### HTML-02: No Markdown Artifacts
**Priority**: High
**Status**: ✅ Complete
**Outcome**: Validated
**Description**: Ensure content doesn't contain markdown code blocks or backticks
**Acceptance Criteria**:
- [x] Final content doesn't contain markdown code blocks (```html or ```)
- [x] Final content doesn't contain backticks wrapping HTML
- [x] Content displays correctly in WordPress editor
- [x] No manual cleanup needed by users
**Notes**: AI often wraps HTML in markdown code blocks that need removal
**Evidence**:
- Complete absence of markdown artifacts in generated content
- Test coverage: test_clean_html_no_markdown

#### HTML-03: HTML Sanitization
**Priority**: Medium
**Status**: ✅ Complete
**Outcome**: Validated
**Description**: Implement HTML sanitization with allowed tags whitelist
**Acceptance Criteria**:
- [x] System sanitizes HTML content before saving
- [x] Whitelist of allowed HTML tags defined (h1-h6, p, strong, em, ul, ol, li, a, img)
- [x] Malicious or unwanted tags removed
- [x] Content structure preserved
**Notes**: Whitelist defined based on WordPress requirements
**Evidence**:
- Only safe HTML tags preserved: h1-h6, p, strong, em, ul, ol, li, a, img
- Only safe attributes preserved: href, src, alt, title
- Disallowed tags removed: script, div, style, iframe, span, etc.
- Test coverage: test_clean_html_preserves_allowed_tags, test_clean_html_removes_disallowed_tags

### Validation

#### VAL-01: Word Count Validation
**Priority**: High
**Status**: ✅ Complete
**Outcome**: Validated
**Description**: Validate actual word count against target word count
**Acceptance Criteria**:
- [x] System calculates actual word count using regex-based HTML stripping
- [x] System compares actual word count to target word count
- [x] Validation uses ±20% tolerance
- [x] System warns user when outside tolerance
- [x] Validation results stored in post document
**Notes**: Used single regex `re.sub(r'<[^>]+>', ' ', html_content)` for HTML tag stripping
**Evidence**:
- WordCountService.count_words(html_content) implemented
- WordCountValidator.validate() implemented
- Comprehensive unit tests for edge cases
- Bug fixed: HTML tag stripping now correctly handles tags with attributes

#### VAL-03: Validation Results Display
**Priority**: Medium
**Status**: ⏳ Deferred to v1.4
**Description**: Display validation results to users
**Acceptance Criteria**:
- [ ] Word count validation results shown in post detail view
- [ ] Section count validation results shown in post detail view
- [ ] HTML cleanliness status shown in post detail view

- [ ] Users can see actual vs target values
**Notes**: Validation is advisory, not blocking

#### VAL-04: Validation Warnings
**Priority**: Medium
**Status**: ⏳ Deferred to v1.4
**Description**: Log warnings when content doesn't meet validation criteria
**Acceptance Criteria**:
- [ ] System logs warning when word count validation fails
- [ ] System logs warning when section count validation fails
- [ ] System logs warning when HTML cleaning removes content
- [ ] Warnings include actual vs target values
- [ ] Warnings stored in job logs for debugging
**Notes**: Validation is non-blocking, content is still saved

#### VAL-05: Tolerance-Based Validation
**Priority**: Medium
**Status**: ⏳ Deferred to v1.4
**Description**: Implement tolerance-based validation instead of strict enforcement
**Acceptance Criteria**:
- [ ] Word count validation uses ±20% tolerance
- [ ] Section count validation uses ±1 section tolerance
- [ ] Validation passes when within tolerance
- [ ] Validation fails when outside tolerance
- [ ] Users can publish content that doesn't meet validation
**Notes**: AI models cannot guarantee exact specifications
